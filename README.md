# fintel-agent

A full-stack starter wired across three tiers:

| Tier | Tech | Owns |
| --- | --- | --- |
| **Frontend** | Next.js 16 (App Router) · Tailwind v4 · shadcn/ui | UI, subscribes to Convex for **live** data |
| **API** | Hono on Bun | **Streaming** AI responses, webhooks, public REST |
| **Data + Auth** | Convex · Convex Auth | Reactive DB, serverless functions, email/password auth |

### The one rule that keeps this clean

> **Convex owns data, queries, mutations, reactivity, and auth. Hono owns streaming, webhooks, and external integrations. Hono is never a pass-through for data the browser could read from Convex directly.**

The browser talks to Convex **directly** for reactive data (live queries), and calls Hono only for the things Convex isn't the right tool for. When Hono itself needs data, it talks to Convex server-to-server via `ConvexHttpClient`.

```
                ┌──────────────────────────┐
   live data    │      Next.js (web)        │   fetch()  ┌──────────────┐
 ┌──────────────┤  React Server Components  ├───────────▶│  Hono (api)  │
 │              │  + Convex React client    │  stream    │   on Bun     │
 │              └──────────────────────────┘            └──────┬───────┘
 │                                                              │ ConvexHttpClient
 ▼                                                              ▼
┌──────────────────────────────────────────────────────────────────────┐
│                      Convex  (DB · functions · auth)                   │
└──────────────────────────────────────────────────────────────────────┘
```

## First-time setup

```bash
# 1. Install everything (from the repo root)
bun install
```

### 2. Provision the Convex backend

This logs you in, creates a dev deployment, generates `convex/_generated/`, and pushes the schema + functions. **Leave it running** (it watches for changes).

```bash
bun run dev:backend          # = `convex dev` in packages/convex
```

On first run it prints your deployment URL, e.g. `https://acme-otter-123.convex.cloud`. Copy it — you'll need it below.

> ℹ️ Until this step runs at least once, TypeScript will report
> `Cannot find module './convex/_generated/*'`. That's expected — the types are
> generated by Convex, not committed.

### 3. Configure Convex Auth (one-time)

In a second terminal, generate the auth keys/env on your deployment:

```bash
cd packages/convex
bun run setup:auth           # = `bunx @convex-dev/auth`
```

This sets `JWT_PRIVATE_KEY`, `JWKS`, and `SITE_URL` on the deployment. When asked for the app URL, use `http://localhost:3000`.

### 4. Create the env files

```bash
# apps/web/.env.local
NEXT_PUBLIC_CONVEX_URL=https://<your-deployment>.convex.cloud
NEXT_PUBLIC_API_URL=http://localhost:8787

# apps/api/.env
PORT=8787
CONVEX_URL=https://<your-deployment>.convex.cloud
WEB_ORIGIN=http://localhost:3000
```

(`*.example` files are provided next to each.)

## Run it

Once Convex has been provisioned once, start all three tiers together:

```bash
bun run dev
```

| Service | URL |
| --- | --- |
| Web (Next.js) | http://localhost:3000 |
| API (Hono) | http://localhost:8787 |
| Convex dashboard | `npx convex dashboard` |

Or run them individually:

```bash
bun run dev:web        # Next.js only
bun run dev:api        # Hono only
bun run dev:backend    # Convex only
```

### Try the demo

1. Open http://localhost:3000 — the landing page pings the Hono API and shows its status.
2. Click **Create an account** → sign up with any email + password (Convex Auth).
3. You land on `/app`:
   - **Live message board** — a reactive Convex query + authenticated mutation. Open the page in two tabs and watch it sync instantly.
   - **Streaming via Hono** — reads a token stream from the Bun service (where real LLM streaming would go).

## Where to build next

- **A new DB table** → add it to [`packages/convex/convex/schema.ts`](packages/convex/convex/schema.ts), then write queries/mutations alongside `messages.ts`.
- **A reactive feature** → call `useQuery(api.x.y)` / `useMutation(api.x.z)` from a client component. Import `api` from `@fintel/backend`.
- **An AI agent / streaming endpoint** → add a route in [`apps/api/src/index.ts`](apps/api/src/index.ts) using `streamText`; swap the demo token loop for the Anthropic SDK. Read it from the browser with `fetch()`.
- **A webhook or public REST endpoint** → also goes in the Hono service; reach Convex with the `ConvexHttpClient` already set up there.
- **More UI components** → `cd apps/web && bunx shadcn@latest add <component>`.
- **OAuth / magic links** → add providers to [`packages/convex/convex/auth.ts`](packages/convex/convex/auth.ts) ([docs](https://labs.convex.dev/auth/config)).

## Deploying

- **Convex**: `cd packages/convex && bun run deploy` (`convex deploy`) → production deployment.
- **Web**: deploy `apps/web` to Vercel; set `NEXT_PUBLIC_CONVEX_URL` (prod) and `NEXT_PUBLIC_API_URL`.
- **API**: deploy `apps/api` to any Bun-capable host (Fly.io, Railway, a container); set `CONVEX_URL` and `WEB_ORIGIN`.
